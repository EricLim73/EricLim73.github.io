Time Tracking

프로그램이 돌아가는 플랫폼마다 시간을 재는 방법이 다르다고 한다. 일단 공통적으로 사용하는 CPU를 보면 제품스펙에 붙는 ~GHz 라는 부분을 보자. 예를 들어 3GHz라고 써있다면 그건 CPU가 1초당 30억 싸이클, 1나노초 마다 3개의 연산을 수행한다는 뜻이다. 참고로 그시간 동안 빛은 30센치정도 움직인다.

프로그램에서 시간을 측정하는 방법은 여러가지가 있다. RDTSC(ReaD Time Stamp Counter)는 인텔칩에서 사용가능한 명령 중 하나다. 해당 명령을 부르면 머신이 작동한 시점부터 얼마나 많은 클럭 싸이클이 지났는지 알려준다. 이걸 측정하고 싶은 코드의 앞과 뒤로 한번씩 불러서 둘의 차이를 구하면 얼마나 많은 싸이클이 지났는지 알 수 있다. 다만 이게 100% 정확히 걸린 “시간"을 알려주는건 아니다. RDTSC는 단순히 클럭 싸이클을 반환한다; 만약 프로그램이 해당 영역을 실행하는 도중 잠시 다른 일을 해야되서 진행을 잠시 멈추는 경우 값이 정확하지 않게 된다. RDTSC를 사용한 측정은 그저 대략적으로 얼마만큼의 싸이클을 쓰는지 아는 정도로 사용하면 좋을거 같다.

GlobalRunning = true;
// __rdtsc()는 intrinsic 함수로써 컴파일 후 어셈블리코드를 보면 따로 call하는 부분없이 바로 rdtsc가 불리는 모습을 볼 수 있다. 
u64 LastCycleCount = __rdtsc();
while (GlobalRunning)
{
    // ...
    // The game loop
    // ...
    u64 EndCycleCount = __rdtsc();
    LastCounter = EndCounter;
}

그렇다면 시간측정은 어떻게 하나? Windows의 경우 QueryPerformanceCounter()라는 함수가 있다. 이는 RDTSC와는 반대로 윈도우가 할 수 있는 최선을 다해서 현재 시각의 Performance Counter를 알려준다. 이 함수도 측정하고 싶은 영역의 앞뒤로 작성하고 차이를 빼면 실행에 사용된 시간차를 알 수 있다.

GlobalRunning = true;
LARGE_INTEGER LastCounter;
QueryPerformanceCounter(&LastCounter);
while (GlobalRunning)
{
    // ...
    // The game loop
    // ...
    LARGE_INTEGER EndCounter;
    QueryPerformanceCounter(&EndCounter);

    // TODO(casey): Display the value here
    // NOTE(EricLim): LARGE_INTEGER의 경우 union으로 된 64비트 데이터다.
    // QuadPart로 접근시 64비트 전체를 접근한다.
    s64 CounterElapsed = EndCounter.QuadPart - LastCounter.QuadPart;
    LastCounter = EndCounter;
}

그럼 이값을 가지고 어떻게 하면 Wall Clock값을 알 수 있을까? 1초에 performance-counter가 얼마만큼 올라가는지를 알면 된다. QueryPerformanceFrequency()가 그 값을 반환하며, 이 값의 경우 머신부트와 동시에 고정된 값으로 유지된다.

int CALLBACK
WinMain(HINSTANCE Instance,
        HINSTANCE PrevInstance,
        LPSTR     CommandLine,
        int       ShowCode)
{
    LARGE_INTEGER PerfCountFrequencyResult;
    QueryPerformanceFrequency(&PerfCountFrequencyResult);
    s64 PerfCountFrequency = PerfCountFrequencyResult.QuadPart;
    // ... 
}

이제 시간을 알아보자. 단순 계산 문제여서 어려운건 없다.

// 1000은 milisec으로 보기 위해서다.
s32 MSPerFrame = (s32)((1000*CounterElapsed) / PerfCountFrequency);

프레임당 걸리는 시간을 알 수 있었다. 60프레임 제한을 둔다면 16밀리초 근처의 값이 나온다. 만약 fps가 알고 싶다면 위 변수들로 계산을 하면 된다. CounterElapsed가 PerfCountFrequency로 나눠 second / frame이 나온걸 보면 CounterElapsed는 counter / frame이라고 알 수 있다. 그럼 여기서 프레임이 분자 시간이 분모가 되게끔 하는 값을 가지고 와서 곱하기 해주면 된다.

s32 FPS = (s32)(PerfCountFrequency / CounterElapsed);

시간을 통한 성능 측정에도 사용할 수 있고 렌더링 프레임을 제한하기 위해 (aka 프레임당 걸리는 시간을 Sleep()같은 기능으로 제한) 사용할 수 도 있다. 물론 프레임 제한의 경우 렌더링 API에서 vertical-flip에 맞춰 스왑체인 넘기는 기능을 설정할 수 있어서 위에 내용을 응용해서 만들 필요가 있나 싶긴하다.

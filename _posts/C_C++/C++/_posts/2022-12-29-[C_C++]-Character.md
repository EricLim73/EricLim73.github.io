# Basic C/C++
    지금까지 배운 C와 C++를 완전 바닥부터 다시 복습 + 공부하고 그걸 기록하기 위한 포스트입니다.  

## Character

> 문자 "1" =\\= 메모리  [ 1 ]

```c
    char c = 1;
    char c1 = "1";

```
둘다 1이 들어가지만 `c`는 `\x`, `c1`은 `1`이 저장된다. 기본적으로 문자는 **`아스키코드`** 전제로 해석된다. 고로 `1`에 해당하는 아스키코드인 `\x`가 입력된다. 반대로 `1`은 아스키코드로 `49`에 해당한다. 디버거로 확인해보면 `char c1 = "1";`에 `c1`은 `49  "1"`이라고 찍힌다. 

<br/>

> 띄어쓰기 =\\= NULL

```c
    char str[3] = "a d";
    // str[0] =     a   (아스키코드 97)  
    // str[1] = [space] (아스키코드 32)
    // str[2] =     b   (아스키코드 38)
```
띄어쓰기도 하나의 문자로 처리한다. 아스키코드를 보면 `SPACE`라고 적힌 항목이 띄어쓰기를 나타낸다. '아무것도 없다'를 나타내는 `NULL`의 경우 문자와 문자열에서 다른 의미를 갖고 있기 때문에 띄어쓰기로 간주하지 않는다. 

<br/>

> ' '  =\\= "  "

```c
    char a = 'a';
    char* b = "bbb";
    char wrong = "wwww";    // => WRONG 
```
`' '` 과 `" "`는 서로 다른 의미를 갖는다. 전자의 경우 ***`문자`*** 를 나타내며, 후자의 경우 ***`문자열`***을 나타낸다. 

<br/>

> NULL == 문자열의 끝

```c
    char* str = "abcd";
```
문자열이라 하면 연속적인 `char`데이터라고 할 수 있다. 이러한 연속적인 데이터의 끝을 가르키기 위해 `NULL`을 사용한다. 문자열 끝에 `NULL`이 붙으면 해당 문자열은 그자리에서 끝난다고 보면 된다. 이러한 이유로 문자엻의 경우 배열로 저장할 시, 혹은 포인터 연산으로 참조할 시 보이는 전체 길이보다 1개 많다고 봐야된다.  

>  C++의 std::string의 경우 NULL값으로 끝나서 생기는 길이 문제는 신경안써도 된다. 해당 클래스의 경우 문자열의 끝을 NULL로 인식하지 않기 때문이다.

## 문자열 저장공간 & 접근

```c
    const char* str = "string";     //  (A)
    char str_Arr[10] = "string";    //  (B)

```
위에 두 라인 모두 정상작동하는 라인들이다. `(A)`의 경우 상수 포인터로 해당 문자열을 받는다. 포인터는 주소를 저장하는 변수다. 이게 정상작동한다는 뜻은 `"string"`이라는 문자열이 자신이 저장된 주소를 반환한다는 뜻이다. 그럼 코드상 작성한 이러한 상수-문자열(`String Literal`)은 어디 저장되는가? 4개의 메모리 영역 중 `ROM (Read Only Memory)`에 저장된다. 이름처럼 읽기만 가능한 영역이다. 그도 당연한게 코드 내용을 저장하고 런타임에 돌리는데 갑자기 해당 메모리가 수정된다는 소리는 런타임에 해당 코드가 마음대로 달라진다는 건데 말이 안되지 않는가. 그래서 포인터로 상수-문자열을 받을 때 `const`를 붙혀서 받는 거다. 실제로 상수-문자열의 반환타입도 `const`다.  
`(B)`의 경우도 위 내용을 바탕으로 해석하자면 `ROM`메모리 어딘가에 저장된 `"string"`이라는 문자열의 저장공간에서 내용을 1바이트씩 복사해서 `str_Arr`배열 칸에 넣겠다는 소리다. 이렇게 하면 배열을 통한 수정을 하더라도 `ROM`에 있는 원본은 영향을 안받기 때문에 문제 없다. 결국 *(A)는 참조, (B)는 복사/붙혀넣기* 라는 소리다

>  배열의 초기화에 있어서 상수문자열을 이용한 초기화는 `char`나 `wchar_t`에만 사용가능하다.  
`uint_8t arr[3] = "abc";` 같은 초기화는 에러처리된다.

## MultiLanguage

```c

    char a[10] = "abc가나다";   // 한글을 1바이트 안으로 표현 불가(다른 나라 언어도 마찬가지)
    wchar_t wc_arr[10] = L"abc한글말";

```

`char`타입이 나타낼 수 있는 1바이트 공간만으로 모든 언어를 나타내기에는 역부족이다. 그래서 많은 사람들이 여러 방법을 고안했다. 그중 가장 많이 사용되는 방식이 `UTF8`형식이다. 그리고 해당 형식을 사용하기 위해 C/C++에서는 `wchar_t`를 사용한다. 해당 타입은 쉽게 말해 `char`의 2배되는 저장공간을 갖는 문자를 저장하기 위한 타입이다. `wchar_t`으로 문자 혹은 문자열을 받을 때에는 앞에 `L`을 붙혀준다. 이렇게 해야 넘겨받는 문자 데이터를 `2바이트`로 인식한다.
